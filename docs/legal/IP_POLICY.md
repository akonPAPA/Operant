# OrderPilot Intellectual Property (IP) Policy

> This document is an internal policy, not formal legal advice. Consult qualified
> counsel before relying on it for a specific legal situation.

## Purpose

This policy describes how OrderPilot's intellectual property is owned, protected, and
shared. It exists to keep founder/company rights clear and to prevent accidental loss
of proprietary value through contributions, public sharing, or demo material.

## What is proprietary OrderPilot IP

The following are treated as proprietary OrderPilot IP:

- Core backend (core-api) and its command/query services, validation/risk engine,
  tenant/security/audit model, and data model
- Frontend dashboard (web-dashboard)
- AI worker and any extraction/OCR/understanding pipelines and prompts
- Bot runtime, channel/messenger integrations, and connector logic
- Integration and ERP/1C handoff design
- Architecture, roadmap, and product/stage documentation
- Prompts, evaluation assets, and internal datasets
- Investor and pitch/demo assets

## Ownership rule

All OrderPilot IP is owned by OrderPilot / [Legal Entity Name]. Creation of code,
design, or documentation within this repository or for this product is intended to
vest ownership in the company, subject to written agreements with contributors.

## Contributions by friends / contractors / employees

Anyone contributing meaningful code, design, or documentation — whether a friend,
contractor, or employee — must have a written IP assignment (and, where relevant, a
work-for-hire or contributor agreement) before or at the time of contribution. Casual
or unpaid help does not create an exception.

## IP assignment requirement

Written IP assignment should be in place before or at the time of any meaningful
contribution. Contributions without a signed assignment create ownership ambiguity
and should be paused until resolved.

## NDA recommendation

A non-disclosure agreement (NDA) is recommended before granting any external party
access to proprietary code, architecture, roadmap, or demo material that reveals
non-public product details.

## Repository access rule

GitHub collaborator access (or any repository access) does not transfer ownership and
does not grant a license to reuse OrderPilot IP outside the scope of authorized work.

## Public sharing rule

Do not paste private repository code publicly (issues, forums, social media, public
gists, AI tools with public/training data settings, or any public channel) without
explicit authorization.

## Demo / pitch material rule

Investor and demo material may show product capabilities, but must not expose secrets,
customer data, credentials, internal endpoints, private architecture details, or
vulnerability details.

## What must not be published

- Secrets, tokens, private keys, or production credentials
- Customer data or customer documents
- Internal endpoints or private infrastructure details
- Private architecture details not intended for public release
- Vulnerability details or unpatched security findings

## Practical checklist

- [ ] Written IP assignment exists for every meaningful contributor.
- [ ] NDA considered before sharing non-public details externally.
- [ ] No private repository code pasted publicly.
- [ ] Demo/screenshots reviewed for secrets, customer data, and internal details.
- [ ] Repository access limited to people with a legitimate need.
