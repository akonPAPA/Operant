# Stage 12C - Operator Quote Review & Exception Cockpit

Stage 12C adds the operator-facing review workflow for draft quotes and channel/document conversion attempts. It is a backend-command workflow, not a demo-only UI: operators can inspect source evidence, validation issues, substitute candidates, approval requirements, and audit history, then apply controlled corrections or route risk to approval.

## Scope

- Review queue for tenant-owned draft quotes and quote conversion attempts.
- Quote review detail with header, source context, conversion attempt, source lines, draft lines, validation issues, substitute candidates, pricing risk, approval requirements, and audit timeline.
- Tenant-scoped validation issue commands: resolve, reject suggestion, apply fix, escalate.
- Controlled quote correction commands for customer, quantity, UOM, product mapping, line removal, and manual follow-up.
- Safe substitute selection/rejection for compatible backend candidates.
- Integration with the existing quote lifecycle and `quote_approval_request` model.
- Audit events for every mutation.

## Non-Goals

- No ERP, 1C, accounting, warehouse, or connector production writes.
- No autonomous quote approval.
- No bot/AI trusted-state mutation.
- No frontend-to-database writes.
- No second approval system.
- No generic task manager.

## Lifecycle Flow

1. Stage 12B creates a `quote_conversion_attempt` and may create a `draft_quote`.
2. `GET /api/v1/quote-review/queue` exposes quotes requiring operator review and their next action.
3. `GET /api/v1/quote-review/{quoteId}` exposes the review detail.
4. Operator commands mutate quote state only through `QuoteReviewService`.
5. Corrections re-run relevant deterministic validation and recalculate quote lifecycle.
6. Approval-required risk is represented by existing `quote_approval_request` records and quote statuses such as `PENDING_APPROVAL`.
7. Existing quote approval endpoints remain the approval authority.

## Issue Resolution Model

Validation issues are quote-owned and tenant-scoped. Commands verify the quote and issue belong to the current tenant and reject terminal quote lifecycle states.

- Resolve marks an issue `RESOLVED` and recalculates lifecycle.
- Reject marks an issue `REJECTED` without pretending the quote is approved.
- Apply fix currently delegates to supported resolve/escalate fix types.
- Escalate marks an issue `ESCALATED`, creates a `QUOTE_REVIEW_ESCALATION` approval request, and moves the quote to `PENDING_APPROVAL`.

## Substitute Selection Behavior

Stage 12C implements substitute selection through the backend.

- Candidates are generated by `ProductSubstitutionService`.
- Candidate product must be compatible with the quote line and tenant.
- Blocked customer substitutes cannot be selected.
- High-risk or approval-required substitutes create a `SUBSTITUTE_POLICY` approval request and move the quote to `PENDING_APPROVAL`.
- Safe substitutes set the line substitute decision and resolve substitute-related issues, then revalidate.

The earlier Stage 12B `selectedSubstituteIds` DTO remains conversion metadata only; substitute selection is now performed by `/api/v1/quote-review/{quoteId}/lines/{lineId}/substitutes/select`.

## Approval Integration

Stage 12C uses `quote_approval_request` and the existing approval state machine. Margin, discount, high-risk substitute, and review escalation risk remain approval requirements. Stage 12C commands never set a quote to `APPROVED`.

## Audit Events

Implemented mutation audit events include:

- `VALIDATION_ISSUE_RESOLVED`
- `VALIDATION_ISSUE_REJECTED`
- `VALIDATION_ISSUE_ESCALATED`
- `QUOTE_LINE_CORRECTED`
- `QUOTE_CUSTOMER_CORRECTED`
- `QUOTE_SUBSTITUTE_SELECTED`
- `QUOTE_SUBSTITUTE_REJECTED`
- `QUOTE_REVALIDATED`
- `QUOTE_REVIEW_READY_FOR_APPROVAL`

Metadata includes tenant id, quote id, line id or issue id when relevant, previous/new value, actor type/user, reason code, note, validation summary, and `externalExecution=DISABLED`.

## Known Limitations

- Assignment is not persisted yet; queue rows expose `assignedOperatorId` as null.
- Quote review source evidence is populated when Stage 12B `quote_source_link` exists; RFQ-created quotes show draft/source state without external source lines.
- Product correction accepts a product id. Product search/picker UX can be improved in a later stage.
